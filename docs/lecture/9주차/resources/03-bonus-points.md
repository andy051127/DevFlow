# 가산점 상세 가이드

가산점은 **자동으로 부여되지 않습니다.** 발표 또는 별도 시간에 **본인이 명시적으로 어필**하고, 검증 가능한 증빙을 제시해야 합니다.

가산점 총합 최대 **+6점**, 발표 평가(30점) 점수에 가산.

## A. AI Agent / 스킬 / 워크플로우 적극 활용 (+1점)

**증빙**:
- 사용한 에이전트, 스킬, 워크플로우 목록과 호출 시점
- 호출로 인해 절약된 시간 / 향상된 품질의 구체 사례 1개 이상

**예시**:
- `@code-reviewer` 서브에이전트로 PR마다 자동 리뷰
- `/spec` `/plan` `/implement` 슬래시 명령으로 단계별 진행
- pre-commit 훅으로 자동 포매팅 / 린트 / 테스트

## B. 본인만의 기법 구성 (+2점) — 가장 큰 가산점

**증빙**:
- 본인이 만든 기법의 **파일/디렉토리**
- "왜 이렇게 만들었는가"를 **2~3분 안에 말로 설명** 가능
- 기법이 실제 프로젝트에 **사용되고 있다**는 흔적 (커밋, 산출물)

**예시 1 — 단일 md 통합 부트스트랩**:
- `AUTHORING.{본인이름}.v{버전}.md` 하나로
  - AGENTS.md, SPECS.md, .github/agents/, .github/skills/,
  - .github/prompts/, .github/instructions/, .github/commands/
  - 모두를 자동 생성하는 도구
- 참조 사례: `AUTHORING.DIMOHY.v1.5.3.md`

**예시 2 — 본인만의 워크플로우**:
- "기획 → 사양 → 계획 → 구현 → 테스트 → 배포" 6단계를
  본인의 슬래시 명령으로 정의

**예시 3 — 본인만의 서브에이전트 패턴**:
- 도메인 특화 서브에이전트 카탈로그를 직접 설계

## C. LLM Wiki 기반 본인 암묵지 운영 (+1점)

**증빙**:
- 본인이 운영하는 노트/위키의 위치 (private repo도 가능)
- 최소 **10개 이상의 항목**
- 최신 LLM 관련 출처(Anthropic / OpenAI / GitHub Copilot Wiki / Cursor Docs 등)에서 발췌·정리한 흔적

**권장 카테고리**:
- 잘 동작한 프롬프트 패턴
- 실패 사례와 원인
- 모델/도구별 차이점
- 본인만의 단축어·트리거

## D. 최신 AI Agent 리포트 발표 (+2점)

**조건**:
- **10분 이상** 발표 (정확히 측정함)
- 최신성: 발표 시점 기준 **최근 6개월 이내** 출시·업데이트된 도구/기법
- 출처 명시 (블로그, 공식 문서, 논문 등)

**권장 주제 예**:
- 새로운 AI Agent 프레임워크 (예: 새로운 CLI 에이전트)
- MCP / Skill / Subagent 생태계의 최신 변화
- 코드 생성 모델의 새 버전 비교
- 실제 회사의 AI Agent 도입 사례

**발표 형태**:
- 별도 시간 신청 (개별 일정 조율)
- 본인의 프로젝트 발표와 분리

## 가산점 신청 방법

1. 본인 리포지토리 내 `BONUS.md` 파일을 만들고
2. 신청하는 항목 (A/B/C/D 중)과 증빙을 정리
3. 최종 발표 시 슬라이드 1장으로 어필
