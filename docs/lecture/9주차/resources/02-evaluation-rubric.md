# 평가 루브릭

**발표 평가 30점 + 과제(결과물) 5점 — 별도 합산.**
가산점은 발표 평가에 최대 +6점.

## 1. 발표 체계성 (10점)

| 세부 | 점수 |
|---|---|
| 시작–본문–마무리 구조의 명확성 | 2 |
| 시간 관리 (배정 시간 ±10% 이내) | 2 |
| 슬라이드의 가독성·일관성·시각자료 활용 | 2 |
| 청중 시선 처리 / 발화 속도 / 톤 | 2 |
| 데모 흐름 (있을 경우) | 2 |

## 2. 질의응답 (8점)

| 세부 | 점수 |
|---|---|
| 즉답 가능성 (모르면 "모릅니다" 인정 포함) | 2 |
| 답변의 깊이 (왜·어떻게까지 설명) | 2 |
| 솔직함 (AI가 만든 부분을 정직히 밝힘) | 2 |
| 질문의 의도를 파악하는 능력 | 2 |

## 3. 개발자 기본 소양 (12점)

| 세부 | 점수 |
|---|---|
| 사용 플랫폼·프레임워크 선택 이유 설명 | 2 |
| 프로젝트 구조(디렉토리/레이어) 설명 | 2 |
| 개발 환경 설정 절차 설명 (zero → run) | 2 |
| 빌드/배포 파이프라인 설명 | 3 |
| 테스트 전략·실행 방법 설명 | 3 |

> 모두 **AI가 만들어줬더라도 본인이 답할 수 있어야** 함.

## 4. 과제(생성 문서) — 별도 5점

> **결과물의 동작 여부로 채점하지 않습니다.**
> AI Agent로 생성·관리한 **프로젝트 문서 세트의 완비도**로 0~5점을 부여합니다.

### 채점 기준 (누적식)

| 점수 | 갖춰진 문서 |
|---|---|
| **0** | 아무것도 없거나, 빈 README 수준 |
| **+1** | `.planning/00-vision.md`, `.planning/01-requirements.md` (기획서/요구사항 MoSCoW) |
| **+2** | 위 + `.planning/02-wbs.md`, `.planning/04-schedule.md` (WBS / 6주 일정) |
| **+3** | 위 + `docs/architecture.md` + `.planning/decisions/ADR-*.md` (최소 3개) |
| **+4** | 위 + `docs/setup.md`, `docs/deploy.md`, `docs/testing.md` (각각 명령어 복붙 가능 수준) |
| **+5** | 위 + `AGENTS.md` + `README.md` 완비 + 본인만의 부트스트랩(예: `AUTHORING.{본인이름}.md`) 또는 `.github/agents`·`.github/prompts` 정착 |

### 검증 방식

- 발표 직전 본인 리포지토리 상태 스냅샷 기준
- 각 문서가 **실제로 채워져 있는가** (자동 생성 후 방치된 빈 파일은 0점)
- 문서들 사이의 **일관성** (예: WBS와 일정이 매칭되는가, ADR이 architecture와 모순되지 않는가)
- AI Agent에게 시킨 흔적 + **본인이 검토한 흔적**(커밋 메시지, 수정 이력)

### 왜 동작이 아닌 문서로 채점하는가

- 본 프로젝트의 핵심은 **"AI Agent로 만든 것을 본인이 책임지고 설명하는 것"**
- 문서가 부실하면 발표·Q&A 점수에서 자연스럽게 깎임
- 동작은 발표 데모에서 평가됨 — 중복 평가 회피
- 7주 안에 동작까지 가지 못해도, **올바른 프로세스를 거친 흔적**은 인정

> 발표 점수와 과제 점수는 **별도 항목으로 합산**됩니다.

## 5. 가산점 (최대 +6점, 발표 평가에 가산)

| 항목 | 가산 |
|---|---|
| AI Agent / 스킬 / 워크플로우 적극 활용 (구체 사례 시연) | +1 |
| **본인만의 기법** 구성 + 설명 (예: 단일 md 부트스트랩) | +2 |
| LLM Wiki 기반 본인 암묵지 운영 증빙 | +1 |
| 최신 AI Agent 리포트 발표 (10분 이상) | +2 |

## 6. 감점 사유

- 발표를 한 사람만 진행 (2인 팀 한정) — **−2점** (발표 평가에서)
- AI 생성물에 대한 핵심 질문에 답변 불가 — **항목별 0점 처리 가능**
- 문서가 AI 자동 생성 후 본인이 한 번도 읽지 않은 흔적 — 과제 점수 단계 하향

## 7. 합산 예시

| 케이스 | 발표(30) | 가산(+6) | 과제(5) | 합 |
|---|---|---|---|---|
| 발표 만점 + 문서 완비 + 가산 만점 | 30 | +6 | 5 | **41** |
| 발표 평균 + 문서 ADR까지 + 가산 일부 | 22 | +3 | 3 | 28 |
| 발표 부진 + 문서만 완비 | 15 | 0 | 5 | 20 |
| 발표 양호 + 문서 거의 없음 | 25 | +1 | 0 | 26 |
