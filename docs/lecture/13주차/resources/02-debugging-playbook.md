# 디버깅 플레이북

## 1. 좋은 버그 리포트 형식

```markdown
## 증상
- 무엇이 일어났는가 (한 줄)

## 재현 절차
1.
2.
3.

## 기대 결과
-

## 실제 결과
-

## 환경
- OS / 디바이스 / OS 버전
- 앱 버전 / 빌드 번호

## 로그 / 스크린샷
(붙임)
```

## 2. 디버깅 5단계

### 1단계: 재현
- 항상 일어나는가, 가끔인가
- 특정 데이터/계정에서만 일어나는가
- → 재현 안 되면 로그/모니터링 추가가 우선

### 2단계: 좁히기
- 마지막으로 동작했던 시점은 언제인가 → `git bisect`
- 어느 레이어인가 — UI? Logic? Data?
- 의심 영역만 격리해서 테스트

### 3단계: 가설
- 원인 후보 3개 적기
- 각각의 검증 방법 1개씩

### 4단계: 검증
- 가장 빠른 가설부터
- **최소 변경**으로 (한 번에 하나만)

### 5단계: 회귀 방지
- 고친 후 **반드시 테스트 추가**
- `lessons/` 에 사례 정리 (가산점 C와 연결)

## 3. AI Agent에게 묻는 법

**나쁜 예**:
> "이 에러 고쳐줘"

**좋은 예**:
> "다음 함수에서 [에러]가 발생합니다.
>  [함수 코드]
>  [에러 메시지]
>  [재현 입력]
>
> 가능한 원인 3가지를 가설로 제시해주세요.
> 각 가설을 검증할 최소 코드 변경도 함께."

## 4. 흔한 모바일 버그 카테고리

| 카테고리 | 첫 의심 |
|---|---|
| 빌드 실패 | 의존성 버전 충돌 |
| 흰 화면 | 비동기 초기화 누락 |
| 스크롤 끊김 | UI 스레드 블로킹 |
| 권한 거절 후 화면 멈춤 | 거절 케이스 분기 누락 |
| 백그라운드 복귀 후 데이터 X | 라이프사이클 핸들러 누락 |
| 메모리 누수 | 리스너 해제 누락 |
| 키보드가 입력 가림 | 스크롤뷰/패딩 처리 |

## 5. 디버깅 사례 기록 템플릿

`lessons/YYYY-MM-DD-짧은제목.md`:

```markdown
## 상황
## 시도 1 (실패)
## 시도 2 (성공)
## 진짜 원인
## 다음에 비슷한 문제 만났을 때
```

→ LLM Wiki 운영(가산점 C)의 핵심 자산
