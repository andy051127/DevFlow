# 데모 시연 체크리스트

## 발표 1일 전

- [ ] 데모 시나리오 종이에 적어두기 (3개)
- [ ] 각 시나리오 처음부터 끝까지 1회 리허설
- [ ] 데모 영상(30초~1분 mp4) 녹화 — 백업
- [ ] 시뮬레이터/에뮬레이터/실기기 모두 준비
- [ ] 화면 미러링 도구 준비 (USB / scrcpy / QuickTime)

## 발표 당일 — 30분 전

- [ ] 노트북 충전 100%
- [ ] 빔프로젝터/디스플레이 연결 테스트
- [ ] 데모 환경 1회 띄워두기 (cold start 회피)
- [ ] Wi-Fi 안 될 때 대비 — 핫스팟
- [ ] 슬라이드 PDF 백업

## 데모 시작 직전

- [ ] 알림 전부 끄기 (메신저, 메일)
- [ ] 화면 잠금 해제 시간 늘리기
- [ ] 전체 화면 모드
- [ ] 마우스 커서 크기 키우기 (가독성)

## 데모 도중 사고 났을 때

| 상황 | 대응 |
|---|---|
| 앱이 크래시 | 재시작 1회만 시도, 안 되면 영상 재생 |
| 인터넷 끊김 | 로컬 모드 시연 또는 영상 |
| 데이터 안 보임 | 미리 만들어둔 더미 계정 |
| 시간이 너무 오래 걸림 | "이 부분은 영상으로" 빠른 전환 |

> **사고 자체보다 대응이 더 평가에 영향을 줍니다.**
