# Q&A 대비 — 자주 묻는 질문 모음

## A. 기획 / 비전

1. 한 줄로 이 프로젝트는 무엇인가요?
2. 누구를 위한 앱인가요?
3. 비슷한 기존 서비스가 있을 텐데, 차별점은?
4. 가장 핵심 기능 하나만 꼽으면?
5. Won't (이번에 안 한 것)는 무엇인가요? 왜?

## B. 기술 선택

6. 왜 [Flutter / RN / Kotlin / Swift] 인가요?
7. 상태관리 라이브러리는 왜 [X]?
8. 백엔드는 왜 [Firebase / Supabase / 직접 구축]?
9. 다른 대안과 비교해본 적 있나요?
10. 만약 다시 시작한다면 다르게 할 것은?

## C. 아키텍처

11. 화면 1개를 그릴 때 데이터는 어디서 어떻게 오나요?
12. API 호출은 어느 레이어에서 하나요?
13. 오프라인 상태에서는 어떻게 동작하나요?
14. 같은 데이터를 여러 화면에서 쓸 때 어떻게 동기화하나요?
15. 화면 추가 시 어떤 파일/디렉토리를 만드나요?

## D. 개발자 기본 소양 (배점 25점!)

16. **환경 설정을 처음부터 어떻게 하나요?**
17. **빌드는 어떤 명령으로 어떻게 하나요?**
18. **배포는 어떤 단계로 진행되나요?**
19. **테스트는 어떻게 작성·실행하나요?**
20. **CI/CD가 있나요? 없다면 왜?**

## E. AI Agent 활용

21. AI Agent를 어떻게 활용했나요?
22. 어떤 부분을 AI가 만들었고, 어떤 부분을 본인이 만들었나요?
23. AI가 만든 코드 중 본인이 수정한 사례가 있나요?
24. 본인만의 AI 활용 기법이 있나요? (가산점 B 기회)
25. AI에게 헛발질당한 사례가 있나요?

## F. 협업 (2인 팀)

26. 작업 분담은 어떻게 했나요?
27. 의사결정이 안 맞았을 때 어떻게 풀었나요?
28. 짝의 코드를 본인이 설명할 수 있나요?

## G. 회고

29. 가장 어려웠던 부분은?
30. 만약 1주일 더 있다면 무엇을 할 건가요?
31. 이 프로젝트로 무엇을 배웠나요?

## 답변 형식 권장

```
[1초 침묵]
"좋은 질문입니다. (X)는 두 가지 측면이 있는데요,
 첫째는 ..., 둘째는 ... 입니다.
 저희는 ... 이유로 [선택]을 했습니다."
```

**모르는 것은**:
> "그 부분은 정확히 모릅니다.
>  지금 추측으로는 ... 이지만, 발표 후 확인해서 알려드리겠습니다."

→ 모르는 걸 인정하는 게 **거짓 답변보다 +점수**.
