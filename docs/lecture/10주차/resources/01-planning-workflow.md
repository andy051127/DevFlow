# 기획 워크플로우 상세

## 0. 시작 전

본인 리포지토리에 빈 디렉토리만 준비:
```
.planning/
.planning/decisions/
docs/
```

## 1. 비전 도출 (시간 약 30분)

AI Agent에게 한 줄 주제만 던지고 **대화로 좁혀나간다**.

**좋은 한 줄 주제 예**:
- "출퇴근길 5분 동안 영어 발음을 교정해주는 앱"
- "동네 단골 카페의 오늘 메뉴를 알려주는 앱"
- "헬스장 운동 기록을 음성으로 입력하는 앱"

**나쁜 한 줄 주제**:
- "AI 앱" — 너무 모호
- "최고의 SNS" — 범위 무한대
- "학원 출석체크" — 동기 약함

> 한 줄 주제는 **누구의 어떤 문제를 어떻게 푸는지**가 들어있어야 합니다.

## 2. 사용자 시나리오 (3개)

```
[누가] 가 [언제/어디서] [무엇을 위해]
앱을 열어서 [어떤 행동]을 하면
[어떤 결과]를 얻는다.
```

이 형태로 3개. AI Agent에게 작성을 시키되, 본인 입으로 다시 말해보세요.

## 3. MoSCoW 분류

| 카테고리 | 정의 |
|---|---|
| **Must** | 없으면 출시할 수 없음 |
| **Should** | 있어야 정상이지만 없어도 데모는 가능 |
| **Could** | 시간 남으면 |
| **Won't** | 이번에는 안 함 (명시적으로 빼기) |

> "Won't"를 명시적으로 적는 게 핵심 — 범위 폭주 방지

## 4. WBS 분해

### 좋은 WBS

```
모바일 앱
├── 사용자
│   ├── 회원가입 화면
│   ├── 로그인 화면
│   └── 프로필 화면
├── 핵심 기능
│   ├── [기능 A] UI
│   ├── [기능 A] 로직
│   └── [기능 A] 데이터 저장
└── 운영
    ├── 빌드 파이프라인
    ├── 배포 스크립트
    └── 분석 / 로깅
```

각 leaf는 1~3일 단위.

## 5. 일정 매핑

WBS의 각 leaf를 6주에 분배:
- 11주차: 환경 + Must의 30%
- 12주차: Must의 50% (중간발표 데모 가능 수준)
- 13주차: Must 완료 + Should 시작
- 14주차: Should 마무리 + 배포

## 6. 검증

이 단계 끝났는지 확인:
- [ ] AI Agent에게 "이 기획을 5문장으로 요약해줘" 했을 때 본인 의도가 맞는가
- [ ] WBS의 모든 leaf가 1~3일로 추정 가능한가
- [ ] 6주 일정이 너무 빡빡하지 않은가 (버퍼 20% 권장)
