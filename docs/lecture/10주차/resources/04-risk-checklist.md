# 위험 체크리스트

각 팀은 최소 5개 이상의 위험을 식별하고 대응 방안을 명시.

## 카테고리별 흔한 위험

### 1. 기술 위험
- [ ] 처음 써보는 라이브러리 / 프레임워크
- [ ] 모바일 권한 (카메라, 위치, 알림) 처리 미숙
- [ ] OS 버전 호환성 (Android 8 vs 14, iOS 14 vs 17)
- [ ] 무료 API 한도 초과
- [ ] 외부 SDK의 갑작스러운 정책 변경

### 2. 일정 위험
- [ ] 시험 기간과 겹치는 주간
- [ ] 다른 과목 과제 마감
- [ ] 명절 / 공휴일
- [ ] 본인의 알바 / 인턴 일정

### 3. 협업 위험 (2인 팀)
- [ ] 의사결정 지연
- [ ] 작업 분담 갈등
- [ ] 한 명이 부진할 때 다른 한 명에게 부담 집중
- [ ] 같은 코드 영역 동시 작업으로 충돌

### 4. AI 의존 위험
- [ ] AI가 만든 코드를 본인이 이해 못함 → Q&A 실패
- [ ] AI가 outdated된 API를 추천
- [ ] AI 사용량 한도 / 토큰 비용
- [ ] AI 도구 자체의 다운타임

### 5. 결과물 위험
- [ ] 데모 시점에 동작 안 함
- [ ] 데모 환경(Wi-Fi, 빔프로젝터) 문제
- [ ] 백엔드 무료 티어 만료

## 대응 방안 형식

```markdown
## 위험: [짧은 제목]
- 카테고리: [기술 / 일정 / 협업 / AI / 결과물]
- 발생 가능성: 상 / 중 / 하
- 영향도: 상 / 중 / 하
- 대응:
  - 예방: ...
  - 발생 시: ...
- 모니터링 지표: ...
```
