# ADR-0003: 데이터 저장 방식 선택

## 상태
확정

## 배경
루틴·일정·달성 기록을 어디에 저장할지 결정해야 한다.
저장 방식은 발표 안정성, 개발 복잡도, 오프라인 동작 여부에 영향을 미친다.

## 검토한 대안
| 방식 | 장점 | 단점 |
|---|---|---|
| SQLite (sqflite) | 오프라인 완전 지원, 외부 의존 없음, 발표 안정 | 기기 간 동기화 불가 |
| Firebase Firestore | 실시간 동기화, 멀티 기기 지원 | 네트워크 의존, 과금 위험, 설정 복잡 |
| Hive | 경량 NoSQL, 빠름 | 복잡한 관계형 데이터 표현에 한계 |

## 결정
**SQLite (sqflite)** 를 우선 선택한다.
Firebase 연동은 Could 항목으로 이후 여유 시 검토한다.

## 이유
- 네트워크·과금·환경변수 없이 동작 → 발표 당일 데모 안정성 최고
- 루틴·일정·달성 기록 간 관계형 데이터 표현에 SQLite가 적합
- `docs/setup.md` 절차가 단순해져 발표 Q&A 방어 용이
- Firebase는 로그인·클라우드 동기화 등 추가 복잡도를 동반하므로
  Must 기능 완성 후 Could 단계에서 검토

## 결과
- 모든 데이터는 기기 로컬 SQLite에 저장
- `data/local/database_helper.dart` 에서 DB 초기화 및 마이그레이션 관리
- Firebase 연동은 Could 항목으로 분류, 이번 버전에서 보장하지 않음
