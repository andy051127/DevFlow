# 요구사항 정의

## 사용자 시나리오

### 시나리오 1 — 주니어 개발자 (루틴 관리)
취업 준비 중인 주니어 개발자가 매일 알고리즘 풀기, 사이드 프로젝트 작업,
기술 블로그 읽기를 루틴으로 등록한다. 매일 체크하면서 스트릭이 쌓이고,
30일 연속 달성 업적을 받아 동기부여가 유지된다.

### 시나리오 2 — 시니어 개발자 (일정 관리)
여러 프로젝트를 병행하는 시니어 개발자가 마감일·발표일 같은 장기 일정을
캘린더에 등록한다. 각 일정을 관련 루틴(코드 리뷰, 문서 작성)과 연결해
일정과 루틴을 한 흐름으로 관리한다.

### 시나리오 3 — 공통 (통계 확인)
한 달 사용 후 통계 화면에서 GitHub 잔디 형태의 히트맵으로 나의 달성
이력을 확인하고, 루틴별 스트릭과 전체 달성률을 통해 어떤 루틴이
약한지 파악한다.

## MoSCoW 기능 목록

### Must — 반드시 있어야 함
| ID | 기능 | 설명 |
|---|---|---|
| M-01 | 루틴 추가·수정·삭제 | 이름, 아이콘/색상, 반복 요일, 목표 설정 |
| M-02 | 루틴 완료 체크 | 오늘 해야 할 루틴 카드 목록, 체크 시 완료 표시 |
| M-03 | 진행률 표시 | 홈 상단 "오늘 N/M 완료" + 프로그레스 바 |
| M-04 | 스트릭 표시 | 루틴별 연속 달성 일수 표시 |
| M-05 | 일정 추가·수정·삭제 | 이름, 날짜, 세부사항 설정 |
| M-06 | 일정 완료 체크 | 캘린더 형태, 체크 시 완료 표시 |
| M-07 | 오늘/내일 일정 요약 | 홈 상단에 가까운 일정 간단 표시 |

### Should — 있으면 좋음
| ID | 기능 | 설명 |
|---|---|---|
| S-01 | 로컬 푸시 알림 | 루틴별 알림 시간 설정, 일정 D-7·D-1 알림 |
| S-02 | GitHub API 연동 | 공개 커밋 이력을 불러와 잔디 형태로 표시 |
| S-03 | 업적 시스템 | 달성 조건별 업적 부여 (예: 7일 연속, 30일 연속) |
| S-04 | 루틴 카테고리 분류 | 학습·사이드 프로젝트·건강 등 카테고리 태그 |
| S-05 | 캘린더 히트맵 | GitHub 잔디 형태로 월별 달성 이력 시각화 |
| S-06 | 루틴-일정 연결 | 루틴 편집 화면에서 관련 일정과 연결 |

### Could — 여유 있으면
| ID | 기능 | 설명 |
|---|---|---|
| C-01 | 다크모드 | 앱 테마 라이트/다크 전환 |
| C-02 | 홈 화면 위젯 | 오늘 루틴 현황을 기기 홈 화면에서 바로 확인 |
| C-03 | 로그인 기능 | 계정 생성·삭제·이름 변경 |
| C-04 | 클라우드 동기화 | Firebase 연동, 기기 간 데이터 공유 |

### Won't — 이번 버전에서 하지 않음
| ID | 기능 | 이유 |
|---|---|---|
| W-01 | 소셜 기능 (친구·랭킹) | 백엔드 복잡도 과다 |
| W-02 | AI 루틴 추천 | 외부 API 의존, 범위 초과 |
