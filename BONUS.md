## B. 본인만의 기법 (+2점)

### 파일
`AUTHORING.Chanhee.md` (프로젝트 루트)

### 핵심 기법 요약
**"초안 → 검토 → 확정" 3단계 워크플로우**

AI에게 바로 작성을 시키지 않고, 초안을 먼저 받아 검토한 뒤
확정하는 방식으로 모든 문서를 작성했습니다.

```
{문서 이름} 초안 작성해줘     ← AI가 초안 생성
       ↓
내가 검토 및 수정 요청
       ↓
초안 그대로 작성해줘          ← 확정 및 파일 저장
```

### 이 기법을 만든 이유
발표 Q&A에서 AI가 만든 내용을 본인이 설명할 수 있어야 하기 때문에,
AI가 만든 것을 그대로 쓰지 않고 반드시 내가 검토·확정하는 구조를
강제하기 위해 설계했습니다.

### 실제 사용된 흔적
- `.planning/` 문서 5개
- `docs/` 문서 4개
- `AGENTS.md`, `README.md`
- ADR 3개
모두 이 워크플로우로 생성·확정되었습니다.
